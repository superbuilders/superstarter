> Persisted Alpha Style design context for the **18seconds** CCAT prep app. This file is the round-opening setup hook from `docs/plans/phase5-master-plan.md` §10 — `teach-alpha-style` was run as the first action of Phase 5 sub-phase 1's commit 1 (`docs/plans/phase5-post-session-review.md` §12 commit 1). Subsequent sub-phases (2-5) inherit this context without re-running setup.

## Project context

- **Product:** 18seconds — a CCAT (Criteria Cognitive Aptitude Test) prep app focused on the 18-second-per-question triage discipline.
- **Surface scope this round:** Phase 5 ships polish + completion on the **non-focus-shell** UX. Specifically: the post-session review surface (sub-phase 1, this commit's foundation), full-length configure page (sub-phase 3), click-to-highlight on the wrong-items browser (sub-phase 4), dojo rename + belt indicator on the post-session summary (sub-phase 5). Plus already-shipped surfaces that get Alpha Style on touch: Mastery Map, drill configure, header chrome, sign-out, navigation, auth screens, error screens, empty states.
- **Focus-shell exclusion is permanent for v1.** The `<FocusShell>` and all its children — chrome row with three bars, `<ItemSlot>`, `<TriagePrompt>`, `<InterQuestionCard>`, audio-tied animations, the dual-bar per-question timer, the blue-to-red half-target color flip, the pace-keyed coloring — are explicitly out of Alpha Style's scope. Any in-question rendering (item prompts, options, immediate post-submit feedback) is excluded for the same reason. This is a load-bearing boundary; do not normalize across it.
- **Brand variant:** This is a partner product (path: `partner/18seconds`), not flagship Alpha marketing. Track flagship Alpha closely (Alpha Style is the design system) but with the product-specific framing — CCAT prep, triage discipline, focus-shell exclusion.

## Design Context

### Users

CCAT test-takers preparing for an Alpha (or Alpha-partner) hiring assessment. Adults. Time-pressured: the test itself is 50 questions in 15 minutes; the prep app's whole proposition is training the user to triage under that pressure. They arrive motivated (the test is gating real outcomes for them) but typically anxious. The post-session surface this round builds is where they see how they're calibrating against the test's pace — the surface should feel like a calm, trustworthy summary rather than a dashboard of metrics. The focus-shell surface (excluded) is where the actual pressure happens; everything outside it is the breathing-room half of the loop.

### Brand Personality

- **Light-first, editorial, polished.** Per master plan §10's Alpha Style anchor. The post-session, Mastery Map, and configure pages should read like calm editorial pages, not a metrics dashboard.
- **Confident and premium without being loud.** Blue and indigo accents are used **deliberately rather than everywhere** — they earn their placement, they don't decorate. Most surfaces are light backgrounds + grayscale typography + occasional accent for the one or two things that matter most on the page.
- **Trust-first.** The user is calibrating themselves against a real test that gates real outcomes. The design's job is not to motivate or hype them — it's to be honest, legible, and fast to scan. No fake gamification, no streak badges out of place, no manipulative dark patterns.
- **Calm next to the focus shell, not contrasting with it.** The focus shell's visual language is tuned for triage pressure (red flips, ticking audio, dual timer bars). Alpha Style surfaces flank the focus shell; they should feel like the user has stepped *out* of the pressure, not like they've stepped into a different product.

### Aesthetic Direction

- **Track flagship Alpha closely.** Alpha Style as published. The 18seconds product is a partner surface, not a redesign opportunity.
- **Editorial typography hierarchy over UI-density.** A post-session screen with five sections (triage score, accuracy, latency, wrong items, strategies) should breathe — generous spacing, clear section breaks, headings that earn their visual weight. Not a stat dashboard.
- **Counts before percentages on the post-session.** Per PRD §6.5: accuracy summary is categorical ✓/✗, no percentages. The single allowed percentage on the post-session is the triage adherence ratio per SPEC §9.7. This isn't an Alpha Style decision per se but it shapes the typography hierarchy: the counts and the threshold-marked latency rows are the primary signal; the triage ratio is the lone numeric.
- **Micro-interactions on non-focus surfaces are welcome; do not bleed into the focus shell.** The `delight` / `animate` operations are appropriate for the post-session's transitions, hover states, button reveals. The focus shell's animations are excluded; never apply Alpha Style motion to a focus-shell child.

### Design Principles

1. **Calm, not motivational.** The user is preparing for a test that gates real outcomes. Dignity over hype. Honest data presentation. No fake encouragement copy, no "great job!" exclamations on bad runs.
2. **The focus shell is a load-bearing exclusion.** Never normalize across it. Alpha Style stops at the focus shell's boundary; the focus shell's tuned visual language stays as-is. If a component lives in `src/components/focus-shell/**` or is rendered as a child of `<FocusShell>`, do not apply Alpha Style operations to it.
3. **Editorial typography over dashboard density.** Post-session, Mastery Map, configure pages: read like a quiet briefing, not a metrics console. Generous whitespace, clear section breaks, headings that carry weight.
4. **Accents earn placement.** Blue and indigo are deliberate, not decorative. The default palette is light background + grayscale typography. Accent on the single most important element of a page or section, not on every interactive surface.
5. **WCAG AA baseline; respect `prefers-reduced-motion` outside the focus shell.** Inferred default — PRD does not specify constraints. The focus shell deliberately uses motion as a triage signal and is exempt; everywhere Alpha Style runs, reduced-motion should suppress decorative animation.
